# AssetTrack - Asset Management System

## Overview

AssetTrack is a comprehensive enterprise asset management system designed to track equipment assignments to employees. The application provides complete lifecycle management for physical assets, including categorization, assignment tracking, status monitoring, and full audit trails. Built as a data-intensive business application, it prioritizes information density, clear workflows, and efficient data entry for managing organizational inventory.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend Architecture

**Framework & UI Library**
- **React 18** with TypeScript for type-safe component development
- **Material-UI (MUI v7)** as the primary component library, chosen for its enterprise-grade data handling capabilities and comprehensive form components
- **Shadcn/UI with Radix UI** primitives also included (dual UI approach with both MUI and Radix components available)
- **Wouter** for client-side routing instead of React Router
- **React Hook Form** with Zod validation for form management
- **TanStack Query (React Query)** for server state management and caching

**Design System**
- Material Design principles focused on information density over visual flair
- Roboto font family (MUI default) with defined type scale
- 8px spacing system with MUI's spacing function
- Dark/light theme support via custom ThemeContext
- Responsive layouts with 12-column grid system
- Maximum content width of 1400px

**State Management**
- Server state: TanStack Query with custom query client configuration
- UI state: React Context (ThemeContext for theme switching)
- Form state: React Hook Form with Zod schema validation
- Authentication state: Custom useAuth hook querying `/api/auth/user`

**Key Features**
- Dashboard with statistics and recent activity
- Asset management (CRUD operations, status tracking, categorization)
- Category management
- Employee (assignee) management
- Assignment tracking with full history
- Search and filtering capabilities
- Pagination on data tables

### Backend Architecture

**Server Framework**
- **Express.js** with TypeScript running on Node.js
- Custom build process using esbuild for server bundling
- Vite for development with HMR support

**API Design**
- RESTful API endpoints under `/api` prefix
- Authentication middleware (`isAuthenticated`) protecting all routes
- Consistent error handling with HTTP status codes
- JSON request/response format
- Request logging middleware for debugging

**Authentication & Session Management**
- **Replit Auth** via OpenID Connect (OIDC)
- **Passport.js** with custom OpenID client strategy
- Session storage in PostgreSQL using `connect-pg-simple`
- Session TTL of 7 days with secure, httpOnly cookies
- User profile stored in database with automatic upsert on login

**Data Layer**
- **Drizzle ORM** for type-safe database queries
- Schema-first approach with shared TypeScript types
- Relational queries using Drizzle relations
- Migration system via `drizzle-kit`

**Core Business Logic (Storage Layer)**
- Abstracted storage interface (`IStorage`) in `server/storage.ts`
- CRUD operations for:
  - Users (required for authentication)
  - Asset Categories (with asset counts)
  - Assets (with category relations)
  - Assignees/Employees (with assignment counts)
  - Assignments (with full relationship details)
- Dashboard statistics aggregation
- Recent activity queries

### Database Schema

**Technology**: PostgreSQL (via `DATABASE_URL` environment variable)

**Core Tables**:
1. **sessions** - Express session storage for authentication
2. **users** - User profiles from Replit Auth (email, firstName, lastName, profileImageUrl)
3. **asset_categories** - Asset categorization (name, description)
4. **assets** - Physical assets (code, name, description, category, status, condition, purchaseDate, purchaseCost, location)
5. **assignees** - Employees/staff (employeeId, firstName, lastName, email, phone, department, position)
6. **assignments** - Asset-to-employee assignments (assetId, assigneeId, assignedDate, returnedDate, status, notes, condition)

**Key Relationships**:
- Assets belong to Categories (many-to-one)
- Assignments link Assets to Assignees (many-to-many through junction)
- Full assignment history maintained (not deleted on return)

**Data Integrity**:
- Generated identity columns for primary keys
- Unique constraints on business identifiers (asset codes, employee IDs, category names)
- Timestamp tracking (createdAt, updatedAt)
- Soft deletes via status flags rather than hard deletes

### Build & Deployment

**Development**
- `npm run dev` - Runs Express with tsx for TypeScript execution
- Vite dev server integrated as Express middleware
- HMR enabled for client-side changes
- Source maps for debugging

**Production Build**
- `npm run build` - Custom build script (`script/build.ts`)
- Vite builds client to `dist/public`
- esbuild bundles server to `dist/index.cjs`
- Selective dependency bundling (allowlist) to reduce cold start times
- CommonJS output format for server

**Configuration Files**:
- `tsconfig.json` - Shared TypeScript config for client/server/shared
- `vite.config.ts` - Client build and dev server configuration
- `drizzle.config.ts` - Database migration configuration
- Path aliases: `@/` (client), `@shared/` (shared types), `@assets/` (static assets)


## Docker Deployment

This project supports **production-ready Docker deployment** using Docker Compose.  
No local Node.js, PostgreSQL, or system dependencies are required — **only Docker**.

---

### Prerequisites

- Docker (v20 or later)
- Docker Compose (v2 or later)
- `.env` file configured (see `.env.example`)

---

### Build & Run

- Production
```bash
docker compose up --build -d
```

- Development
```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml up --build
```

- Migrations
```bash
docker compose run --rm migrate
```

- Database backup
```bash
docker exec [pg_container] pg_dump -U postgres [database_name] > backup.sql
```

- Database restore
```bash
docker exec [pg_container] pg_dump -U postgres [database_name] > backup.sql
```
